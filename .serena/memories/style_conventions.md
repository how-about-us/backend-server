# Style And Conventions\n- Follow `AGENTS.md` and `CONTRIBUTING.md` before making changes.\n- Use Lombok actively; prefer `@RequiredArgsConstructor` for constructor injection.\n- Do not introduce interfaces when there is only one implementation.\n- Do not use ternary operators.\n- Always use braces for `if`, `for`, and `while`, even for one-line bodies.\n- Keep code depth at 2 or less; simplify with guard clauses or extracted methods when needed.\n- Use imports instead of fully qualified class names in code.\n- Package layout is domain-oriented: `controller`, `service`, `repository`, `entity`, while cross-cutting concerns live under `common`.\n- DTOs commonly use Java `record`; controllers and services use annotation-driven Spring style with OpenAPI annotations and Bean Validation.\n- Business exceptions should use custom exception types and be handled in `GlobalExceptionHandler`.\n- Input shape validation belongs at the request DTO/controller boundary; service layer should focus on business rule validation.
