# AI Docs Guide

`docs/ai/`는 이 저장소의 기능, API, 데이터 구조, 설계 결정을 관리하는 로컬 기준 문서 모음이다.

## Priority

문서를 해석하거나 수정할 때는 아래 우선순위를 따른다.

1. 사용자 최신 요청
2. 실제 코드와 테스트
3. `docs/ai/*.md`
4. `AGENTS.md`
5. 외부 문서와 메모

우선순위가 높은 정보와 낮은 정보가 충돌하면 낮은 쪽을 그대로 따르지 말고 먼저 불일치를 보고한다.

## Document Roles

- `features.md`: 무엇을 만들어야 하는지, 어떤 기능이 있고 어떤 기능이 미결인지 관리한다.
- `erd.md`: 저장해야 하는 데이터 구조와 관계, 제약조건을 관리한다.
- `decisions/`: 설계 결정과 그 이유, 영향 범위를 짧게 기록한다.

## Status Labels

문서에는 아래 상태 표현을 일관되게 사용한다.

- `초안`: 구조만 정리된 상태. 구현 과정에서 충분히 변경될 수 있다.
- `예상안`: 합리적인 가정이나 제안이지만 아직 팀에서 확정하지 않았다.
- `확정`: 현재 구현 기준 또는 팀 합의 기준으로 사용 가능한 상태다.
- `미결`: 의사결정이 끝나지 않아 후속 확인이 필요한 상태다.

## Update Rules

갱신 규칙은 `AGENTS.md`의 **Doc Update Rules** 섹션을 따른다.

## Writing Guidance

- 각 문서는 자기 책임 범위 안의 사실만 유지한다.
- 같은 사실을 여러 문서에 중복으로 길게 적지 않는다.
- 아직 확정되지 않은 내용은 확정된 사실처럼 쓰지 않는다.
- 코드와 문서가 어긋나면 문서를 조용히 믿고 진행하지 말고 먼저 불일치를 보고한다.
