# FE-001-02: 토큰 재발급 (Refresh)

## 메타
- 상태: 대기
- 담당: backend/박주영
- 우선순위: P1
- Notion ID: TBD
- 관련 저장소: Redis (`refresh:{userId}`)

## 개요

Access Token 만료 시 Refresh Token으로 새 Access Token을 발급한다. Refresh Token Rotation 적용.

---

## API 명세

### `POST /auth/refresh`

**Request Body**
```json
{
  "refreshToken": "uuid-string"
}
```

**Response `200`**
```json
{
  "accessToken": "jwt-string",
  "refreshToken": "new-uuid-string"
}
```

**Error Cases**
- `401` — Refresh Token 없음 / 만료 / Redis 불일치

---

## 구현 노트

- Redis에서 `refresh:{userId}` 키로 저장된 토큰과 요청값 일치 여부 검증
- 검증 성공 시: 새 Access Token 발급 + 새 Refresh Token 발급 후 Redis 갱신 (Rotation)
- 기존 Refresh Token은 즉시 삭제 (재사용 방지)
- Redis TTL은 Refresh Token 만료와 동일하게 설정 (7일)

---

## DoD

- [ ] 유효한 Refresh Token으로 새 Access Token + Refresh Token이 발급된다
- [ ] 만료되거나 잘못된 Refresh Token은 `401` 반환한다
- [ ] 한 번 사용한 Refresh Token은 재사용 불가하다 (Rotation)
