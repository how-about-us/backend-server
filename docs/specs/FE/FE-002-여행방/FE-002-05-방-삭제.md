# FE-002-05: 방 삭제

## 메타
- 상태: 대기
- 담당: backend/박주영
- 우선순위: P2
- Notion ID: TBD
- 관련 테이블: `rooms`, `room_members`, 연관 테이블 전체

## 개요

방을 삭제한다. ADMIN만 가능. 삭제 정책은 미결.

---

## API 명세

### `DELETE /rooms/{roomId}`

**Header**: `Authorization: Bearer {accessToken}`

**Response `204`** — No Content

**Error Cases**
- `403` — ADMIN이 아닌 경우
- `404` — 존재하지 않는 방

---

## 구현 노트

> **미결:** soft delete vs hard delete + CASCADE  
> soft delete: `deleted_at` 컬럼 추가, 쿼리마다 `WHERE deleted_at IS NULL` 필터 필요  
> hard delete: DB CASCADE로 연관 데이터 일괄 삭제 (단순하지만 복구 불가)

- MVP에서는 hard delete + DB-level CASCADE로 구현
- 삭제 전 ADMIN 권한 검증 필수
