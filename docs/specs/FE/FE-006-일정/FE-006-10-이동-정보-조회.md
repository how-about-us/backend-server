# FE-006-10: 이동 정보 조회

## 메타
- 상태: 대기
- 담당: backend/박주영
- 우선순위: P2
- Notion ID: TBD
- 관련 테이블: `schedule_items`

## 개요

비동기로 계산된 이전 장소와의 이동 거리·시간·수단을 조회한다.

---

## API 명세

이동 정보는 FE-006-05 항목 목록 조회 응답에 포함되므로 별도 엔드포인트 불필요.

단, WS 이벤트로 이동 정보 갱신을 실시간 수신:

**STOMP SUBSCRIBE** `/topic/rooms/{roomId}/schedule`
```json
{
  "event": "TRAVEL_INFO_UPDATED",
  "itemId": 5,
  "travelFromPrev": {
    "distanceMeters": 5000,
    "durationSeconds": 600,
    "travelMode": "DRIVING"
  }
}
```

---

## 구현 노트

- `@Async` 메서드에서 Google Routes API 호출 완료 후 DB 저장 → WS 이벤트 발행
- 이동 정보 없는 경우(첫 항목 또는 미계산): `travelFromPrev: null`
