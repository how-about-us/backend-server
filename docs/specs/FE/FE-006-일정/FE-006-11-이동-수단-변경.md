# FE-006-11: 이동 수단 변경

## 메타
- 상태: 대기
- 담당: backend/박주영
- 우선순위: P2
- Notion ID: TBD
- 관련 테이블: `schedule_items`

## 개요

이동 수단(`travel_mode`)을 수동으로 변경하면 Routes API를 재호출해 이동 정보를 갱신한다.

---

## API 명세

### `PATCH /rooms/{roomId}/schedules/{scheduleId}/items/{itemId}/travel-mode`

**Header**: `Authorization: Bearer {accessToken}`

**Request Body**
```json
{
  "travelMode": "TRANSIT"
}
```

**Response `202`** — Accepted (비동기 처리 중)

---

## 구현 노트

- `travelMode` Enum: `DRIVING`, `WALKING`, `TRANSIT`, `BICYCLING`
- 수락 즉시 `202` 응답 → `@Async`로 Routes API 재호출 → DB 갱신 → WS 이벤트 발행 (FE-006-10 참고)
- 이전 항목이 없는 첫 번째 항목은 이동 정보 없음 → `400` 반환

---

## DoD

- [ ] `travel_mode` 변경 후 Routes API 재호출이 발생한다
- [ ] 재계산된 이동 정보가 WS로 수신된다
- [ ] 첫 번째 항목 변경 시도 시 `400` 반환된다
