# FE-007-03: 재접속 시 미수신 메시지 동기화

## 메타
- 상태: 대기
- 담당: backend/박주영
- 우선순위: P2
- Notion ID: TBD
- 관련 테이블: `messages`

## 개요

WS 재연결 시 마지막으로 수신한 `messageId` 이후의 메시지를 조회해 동기화한다.

---

## API 명세

### `GET /rooms/{roomId}/messages?after={lastMessageId}`

> FE-007-02 목록 조회 엔드포인트와 동일. `after` 파라미터로 미수신 메시지만 조회.

**Query Params**
- `after`: 마지막 수신 `messageId` (해당 ID 이후 메시지만 반환)

**Response `200`**
```json
{
  "messages": [...],
  "hasMore": false
}
```

---

## 구현 노트

- `WHERE id > :after ORDER BY id ASC` 쿼리
- 클라이언트는 WS 재연결 시 로컬에 저장된 마지막 `messageId`를 `after`로 전달
- 미수신 메시지가 많은 경우 pagination 필요 (`hasMore: true`)
